# 03 设计规范

## 视觉语言

- 整体采用黑白单色系。
- 强调“黑底、白字、少量灰阶”。
- 页面要有明显的档案感、工作台感和音乐工具感。

## 布局要求

1. 头部
   - 标题压缩成一行。
   - 关键信息尽量集中。

2. 中部
   - 主要承载生成输入和生成结果。
   - 不要堆太多模块，避免页面过长。

3. 底部
   - 唱出来、字幕、历史集中放在一起。
   - 以横向或紧凑分栏方式呈现。

## 交互要求

- 每个小标题都要有悬停说明。
- 设置里的内容尽量用卡片、标签页或抽屉方式控制显隐。
- 当前状态要有明确反馈，不能让用户猜。

## 组件要求

- 按钮要有清晰主次。
- 下拉、输入、自定义字段尽量同栏组合。
- 音乐字幕要像轨道一样可阅读、可点击、可追踪。

## 可用性要求

- 重要按钮要在同一视线范围内。
- 播放、下载、跳转不能藏得太深。
- 历史区要能快速筛选和回看。
