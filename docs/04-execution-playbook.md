# 04 执行步骤

## 每次改功能前

1. 先看 `docs/01-requirements.md`，确认需求边界。
2. 再看 `docs/02-technical-spec.md`，确认状态、接口、存储方式。
3. 再看 `docs/03-design-spec.md`，确认页面位置和交互方式。
4. 再开始改代码。

## 每次改功能时

1. 先做最小可用版本。
2. 再补进度、状态、异常提示。
3. 再补历史、下载、回放等收尾功能。
4. 最后做一次页面整理和布局压缩。

## 每次改完后

1. 检查脚本语法。
2. 在本地页面跑一遍关键流程。
3. 确认空状态、成功状态、失败状态都能看到。
4. 记录新增的行为变化，必要时同步更新文档。

## 推荐验收流程

- 输入一段内容生成歌词。
- 点击生成并唱出来。
- 播放、暂停、点击字幕跳转。
- 下载音频。
- 在历史里找到刚才那条记录，继续播放、下载、删除。
