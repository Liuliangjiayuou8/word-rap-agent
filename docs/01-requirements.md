# 01 需求规范

## 项目目标

做一个“一词成歌”的 RAP 智能体，让用户输入一个词、一句话、一首诗或一段文稿后，自动生成可唱的说唱内容，并支持播放、下载、历史管理和字幕跳转。

## 核心功能

1. 输入生成
   - 支持单词、短句、诗歌、文稿输入。
   - 支持风格、情绪、炸点强度、篇幅、人声设定等参数。

2. 生成与播放
   - 支持本地生成。
   - 支持 MiniMax 音乐生成。
   - 支持播放、暂停、下载音频。
   - 支持空格键控制播放与暂停。

3. 历史管理
   - 仅在“生成并唱出来”成功后保存结果。
   - 支持查看、播放、下载、删除、筛选。

4. 设置管理
   - API 接入放在设置里。
   - 生成历史放在设置里提供跳转或查看入口。
   - 风格、情绪、人声等要可快速选择，也要支持自定义。

5. 对外分享
   - 支持部署为公网网页。
   - 朋友可直接通过链接访问。

## 非目标

- 不在前端保存任何不必要的长期密钥。
- 不做复杂账号系统。
- 不把页面做成多页面站点，优先单页完成主流程。

## 验收标准

- 输入后能生成结果。
- 生成结果能播放、下载、回看。
- 空格键能控制播放暂停。
- 历史数据刷新后仍能保留。
- GitHub 仓库首页需要能说明项目用途、部署方式和在线地址。
