# 02 技术规范

## 页面结构

- 主区域：输入、参数、生成结果、提示词预览。
- 底部区域：唱出来、字幕、生成历史。
- 设置层：API 接入、历史入口、辅助配置。

## 状态原则

1. 生成状态要明确区分：
   - 待命
   - 生成中
   - 成功
   - 失败

2. 字幕状态要能跟随音频进度同步。

3. 播放状态与历史播放共用同一套音频播放器和字幕轨道。

## 接口原则

- 优先兼容 OpenAI 风格接口。
- MiniMax 相关能力优先按官方文档字段处理。
- 遇到模型或套餐不匹配时，要有回退逻辑，不允许整条链路直接断掉。

## 存储原则

- 设置使用本地持久化。
- 历史记录使用本地持久化。
- 音频若可缓存，尽量缓存到本地浏览器可读存储。

## 交互原则

- 键盘快捷键不得影响输入框编辑。
- 点击字幕必须可跳转。
- 点击历史必须能恢复对应播放上下文。

## 代码原则

- 一个功能尽量复用同一套状态流。
- 新功能优先补在现有单页里，不轻易拆分文件。
- 每次改动后都要做语法校验和页面验证。
